[#{cl_sst_sqlite_database_deserialize_fttcexwfwu_id}]
= The `{cl_sst_sqlite_database_deserialize_fttcexwfwu_name3}` function (overload {fttcexwfwu_overload})

[source,cpp,subs="{sst_subs_source}"]
----
include::001_fttcexwfwu/decl.adoc[]
----

The {cl_sst_sqlite_database_deserialize_fttcexwfwu_chop3_prose2}
attaches the `schema` database in `*this` to the serialized database
that consists of the `size` bytes pointed to by `buffer`.
The attached database will be read-only.

If the `schema` database is the `temp` database, then an exception will
be thrown, as the `temp` database cannot be deserialized into.

If the `schema` database already exists, then it will be detached before
attaching it to the serialized database.
This will occur even if the `schema` database is the `main` database,
which cannot normally be detached.

If the buffer is modified or the lifetime of the buffer ends before the
lifetime of the attachment ends, then the behavior is undefined.
In other words, the caller must ensure that the buffer continues to
exist and is not modified while the attachment continues to exist.
